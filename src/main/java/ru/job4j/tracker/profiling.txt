1) При запуске StartUI JVM использует SC G1, выделяя 4,3Гб под Heap. Сначала задействуется примерно 14Мб памяти, но
примерно каждые 30 секунд прибавляется по примерно 2Мб, через несколько минут происходит GC и количество используемой
памяти падает ниже 5Мб. Далее процесс повторяется. Количество загруженных классов примерно 2300, растет очень медленно.
Количество потоков после запуска поднялось до 17, но потом стабилизировалось на 14. Одна процедура GC занимает 0.003
секунды. Через какое-то время после старта G1 перемещает созданные классы в "G1 Old Gen", "G1 Eden Space",
 "G1 Survivor Space" остаются практически пустыми.
2) При создании новой заявки график памяти Heap не изменился (ступенчатый рост со временем). Было одномоментно создано
примерно 80 новых классов. Второй запуск G1 произошел примерно через 14 минут после старта приложения (задействованная
память упала с 32Мб до 4Мб).
3) При редактировании заявки особых изменений не наблюдается.
4) При добавлении нескольких новых заявок никаких изменений в динамике работы программы не наблюдается. Количество
созданных классов не меняется.
5) При поиске по id и имени никаких изменений не наблюдается.
6) При удалении заявки никаких изменений не наблюдается.

В итоге количество загруженных классов остановилось на 2397 и не меняется при использовании программы (создании, поиске,
редактировании заявок). График задействованной памяти выглядит в виде лестницы, идущей вверх, где каждый новый пик
выше предыдущего. Количество используемых потоков остановилось на 14 (максимальное значение после
запуска было 17).

За время наблюдения G1 запускался 2 раз (оба по 0,003 секунды).
Профилирование проводилось с помощью утилиты gconsole.

Еще одно профилирование с добавлением опций: добавить много заявок (можно указать количество) и удалить все заявки с
данным именем.
1) Добавление 1млн. заявок дает прибавку в занимаемой памяти примерно 110Мб, запускается G1. При первом добавлении было
5 запусков, при каждом последующем 2. Среднее время задержки 0,02 секунды.
2) Запуск удаления 1млн. заявок не приводит к изменениям в показателях памяти. G1 не запускается. Сам процесс занимает
слишком много времени (был вручную прерван).
3) Попытка создания 1млрд. заявок вызвала очень быструю загрузку всего Heap (4.3Гб), G1 Young Generation работал
 63 раза (всего 2,389 секунд), а G1 Old Generation 2 раза (всего 2,611 секунд). Таким образом, G1 Old Generation делал
одну очистку в примерно 34.4 раза дольше, чем G1 Young Generation. Через примерно минуту программа выдала
OutOfMemoryError: Java heap space.

Профилирование проводилось с помощью утилиты gconsole.
